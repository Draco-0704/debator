2. ARCHITECTURE OVERVIEW
----------------------------------------------------------------

User submits news article
         |
         v
   [Claim Extractor]          <-- spaCy NER + Groq LLM
         |
    _____|_____
    |         |
    v         v
[Verifier]  [Falsifier]       <-- Both search web independently
 Agent       Agent
    |         |
 Evidence   Evidence
    |_________|
         |
         v
   [Judge Agent]              <-- Weighs both sides, gives verdict
         |
         v
  Final Verdict + Report

4. BUILD PHASES
----------------------------------------------------------------

PHASE 1 — NLP Foundation (Day 1-2)
------------------------------------
- Set up spaCy for Named Entity Recognition (who, where, what)
- Integrate HuggingFace cross-encoder/nli-deberta-v3-small
  for stance detection
- Build claim_extractor.py:
    Input  -> raw news article
    Output -> 3 to 5 key factual claims

PHASE 2 — Tool Layer (Day 2-3)
--------------------------------
- Build DuckDuckGo search wrapper
    Library: duckduckgo-search (no API key required)
    Per claim: fetch top 5 evidence snippets from web
- Build stance scorer:
    Does this evidence SUPPORT or CONTRADICT the claim?
    Uses NLI model to score each snippet

PHASE 3 — Agent Logic (Day 3-5)
---------------------------------
- Verifier Agent:
    * Searches for supporting evidence per claim
    * Builds a "pro-truth" case using Groq LLM
    * System prompt: "You are trying to VERIFY this claim"

- Falsifier Agent:
    * Searches for contradicting evidence per claim
    * Builds a "counter-truth" case using Groq LLM
    * System prompt: "You are trying to FALSIFY this claim"

- Both agents use identical tools but opposite objectives

PHASE 4 — Judge Agent (Day 5-6)
---------------------------------
- Receives full report from Verifier + Falsifier
- Scores each claim independently:
    SUPPORTED / REFUTED / UNVERIFIABLE
- Outputs final article verdict:
    REAL / FAKE / MISLEADING  +  confidence percentage
- Produces a human-readable reasoning paragraph
- This transcript = built-in explainability

PHASE 5 — Streamlit UI (Day 6-7)
----------------------------------
- Text input box: paste article text or URL
- Live debate display:
    Left panel  -> Verifier arguments
    Right panel -> Falsifier arguments
- Final verdict card:
    Green  = REAL
    Red    = FAKE
    Yellow = MISLEADING
- Deploy to Streamlit Community Cloud (free, permanent URL)

7. RESEARCH PAPER CONTRIBUTIONS
----------------------------------------------------------------

As you build, document these as your novel contributions:

1. Adversarial Debate Framing
   - Novel over single-model classifiers
   - Two agents with opposing objectives = richer signal

2. Claim-Level Granularity
   - Verdict given per individual claim, not just per article
   - More interpretable and trustworthy outputs

3. Stance Detection as Evidence Scoring
   - NLI model repurposed for ranking web evidence quality
   - Bridges retrieval and reasoning

4. Built-in Explainability
   - The debate transcript IS the explanation
   - No need for post-hoc XAI — transparency is structural

Paper Title Suggestion:
"Adversarial Multi-Agent Debate for Automated Misinformation Detection"

8. FREE TIER LIMITATIONS & WORKAROUNDS
----------------------------------------------------------------

Problem                    | Workaround
---------------------------|-----------------------------------
Groq rate limits           | Add time.sleep(1) between calls
DuckDuckGo blocking        | Rotate to Serper.dev as fallback
HuggingFace cold start     | Use local spaCy model as backup
Streamlit session timeout  | Cache results with @st.cache_data
